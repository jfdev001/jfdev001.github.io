# Git for Everyone

Version control is an essential tool for modern software development and
research, yet many people either do not use it or use only a small subset of
its capabilities. This workshop introduces the fundamentals of Git and
collaborative workflows using GitHub. Participants will learn the core concepts
behind version control, including commits, branches, and repositories, and how
these ideas support effective collaboration and reproducible work. The session
combines short explanations with hands-on exercises using a small,
Fortran-based weather simulation project. Participants will practice common
workflows such as tracking changes, creating branches, opening issues, and
contributing through pull requests. By the end of the workshop, attendees will
have a practical mental model of Git and GitHub that they can apply to their
own projects. No prior Git experience is required. Participants are encouraged
to set up Git and create a GitHub account before attending the session so that
they can complete the hands-on exercises during the session. A link to the
slides and materials will be made available to participants in the week prior
to the session. Participants are encouraged to look at the first few slides in
order to determine if the session is a good fit for them.

You can open the slides locally by cloning this repository and opening the
`index.html` file. Alternatively, you can install `quarto` and do

```shell quarto render git-for-everyone.md ```
